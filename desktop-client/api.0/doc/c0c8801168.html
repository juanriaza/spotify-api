Class: Application

api/models~ Application

new Application

An application object that handles your application's interaction with the rest of the Spotify client. Please note that you should *not* instantiate this class directly - instead, use the models.application property.

Members

observe

Register an observer. The Application object fires the following events:
  • EVENT.ACTIVATE: Fires when the application is activated.
  • EVENT.DEACTIVATE: Fires when the application is deactivated.
  • EVENTS.ARGUMENTSCHANGED: Called when the arguments property changes.
  • EVENT.LINKSCHANGED: Called when the links property changes.
Source:
  • models.js, line 452
See:

Example
var sp = getSpotifyApi();
var models = sp.require('$api/models');

models.application.observe(models.EVENT.ACTIVATE, function() {
  console.log('Application activated!');
});

activate

Activates the application view, so that it is the focus of keyboard events. Call this method only in response to user actions, such if user further input is required when dragging a track onto the application.

Shows the standard Spotify "Share" popup to allow the user to share the given URI with a variety of social networks.
Parameters:
Name Type Description
