Class: Cache

Cache

new Cache

All "metadata items" (Album, Artist, Playlist, Track and URI) are cached on JavaScript side, because it is quite expensive to cross the bridge each time one is needed. The cached items may not contain no metadata, some metadata or all metadata. The items are never evicted from the cache, but a longer term plan is to implement that though, so the user of the API cannot depend on the fact that Item.fromURI always returns the same item for a given URI.
