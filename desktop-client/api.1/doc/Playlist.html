Class: Playlist

Playlist

The playlist object represents a playlist owned by a certain Spotify user.

new Playlist

Never construct a playlist object using the default constructor - use fromURI() instead.
Properties:
Name Type Description
allows object A dictionary of the allowed operations on the playlist.
collaborative boolean Indicates if this is a collaborative playlist.
description String A description for the playlist. This is a human readable string that can be presented to the user. Make sure to call the proper string decoding method before using the string in the DOM.
name String The name of the playlist. This is a human readable string that can be presented to the user. Make sure to call the proper string decoding method before using the string in the DOM.
image String The image URI for the playlist. The format of the image URI is platform dependent, but will always be something that can be used as the source of an img element or a background-image in CSS. The size of the image is not defined and might differ between platforms, so it is recommended that applications use the imageForSize method to get an appropriately sized image. In general, it's best to use the Image view from the views framework to display images - it will load the image in the background (while displaying a placeholder) and make sure to pick the correct size "intelligently", based on the current screen resolution and available bandwidth.
owner User The owner of a playlist. Depending on the privacy settings of the Spotify client, this may or may not be empty.
published boolean Indicates if the playlist belongs to the currently logged in user's list of public playlists.
subscribed boolean Indicates if the the currently logged in user is subscribed to this playlist.
subscribers Collection A collection of users that are currently subscribed to this playlist.
tracks Collection The tracks in this playlist.
uri String The URI of the playlist.

createTemporary

Creates a temporary playlist that can be used as the context to play tracks in. This is useful to know when it is your app that is responsible for the currently playing track, or if the user decided to start playing music from some other part of Spotify. Please note that your app is never allowed to override the currently playing track, unless in direct response to an action initated by the user, e.g., clicking a track.
Parameters:
Name Type Description
name string The name of the temporary playlist. This can be used as an identifier by your app for whatever purpose. It is never displayed to the user in the regular Spotify interface.
