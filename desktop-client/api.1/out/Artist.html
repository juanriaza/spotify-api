Class: Artist

Artist

The artist object represents an artist in the Spotify catalogue.

new Artist

Never construct an artist object using the default constructor - use fromURI() instead.
Properties:
Name Type Description
albums Collection The artist's own albums.
appearances Collection Albums on which the the artist appears.
biography String The biography associated with the artist.
genres Collection The genres associated with the artist.
name String The name of the artist. This is a human readable string that can be presented to the user. Make sure to call the proper string decoding method before using the string in the DOM.
popularity number The popularity rating of the artist. This is a value between 0 and 100, inclusive, with 0 meaning a very low popularity and 100 a very high popularity.
portraits Collection Portraits (images) of the artist.
related Collection Other artists that somehow relates to this artist.
singles Collection The artist's own singles.
uri String The URI of the album.
years object The years during which the artist was/is active. This value of the property has two fields: "from" and "to".
user User The user associated to the artist, if any exists.

Methods

fromURI

Returns the artist for a given Spotify URI. The URI is the only property that is loaded on the artist immediately.
Parameters:
Name Type Description
uri String The Spotify URI of the artist.
Since:
  • 1.0.0
Source:

Returns:
The artist matching the given URI.
Type
Artist
Example
models.Artist.fromURI('spotify:artist:5bWRCM3vFGqamlNxSzNj1O').load('name').done(function(artist) {
  console.log(artist.uri + ': ' + artist.name.decodeForText());
  document.getElementById('artist').innerHTML = artist.name.decodeForHtml();
});
