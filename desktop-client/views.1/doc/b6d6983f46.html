Class: List

views/list~ List

Create a new list of items from a collection. Not all items from the collection will be added directly. It uses snapshots, and will partially fill when you call list.init(). If you want more items, you call list.more() which will get a new round with the right offset. Call one of the factory methods to create an instance of List.

new List

Use a factory method to instantiate.

<static> availableOptions

These are the available options for the list view.
Properties:
Name Type Description
numItems number Number of items to get. If not set, the list will load the items on the fly.
fetch string The way to fetch items.
  • 'once': Only fetch initial items to fill the view, nothing more.
  • 'scroll': (Default) Fetch initial items, and then more when scrolling.
  • 'greedy': Fetch all items at load time. Do NOT use 'greedy' for searches, as it is really expensive. Suitable for albums, since they are usually short.
numBuckets number Number of buckets of items to get initiallly on load. One bucket is the number of items that fit into the view at once. Default: 3
header string If and how the header will be displayed.
  • 'no': Don't show any header.
  • 'yes': Show a header.
  • 'fixed': (Default) Show a header and keep it fixed to the top of the viewport when scrolling past the top of the list.
type string The type of list to get.
  • 'tracks': (Default) The most common list, and the most supported one.
  • 'albums': List of albums, not all fields are supported for this.
  • 'artists': List of artists, not all fields are supported for this.
layout string The layout to use.
  • 'default': The default table list layout.
  • 'toplist': Toplist layout.
getItem function Function that will return a DOM node for each list item. The function is passed two arguments: the item and the index for that item. Defaults to a predefined function.
fields Array The fields to use in the list. These are the available fields:
  • 'track': The track name.
  • 'artist': The artist name (can be multiple artists), rendered as links to the artist page.
  • 'album': The album name, rendered as a link to the album page.
  • 'trackartist': The track name together with the artist name. Can be used for a list of tracks for an album. If the track artist differ from the album artist, the artist will also be shown in this field. If the track artist is the same, the track name will take up all of the space in this field for that row.
  • 'time': The duration of the track.
  • 'popularity': The popularity of the track (also works for albums and artists lists).
  • 'image': The image of the track (also works for albums and artists lists).
  • 'number': The track number within its album.
  • 'ordinal': The number within the current list.
  • 'nowplaying': An icon when the row is playing, empty otherwise.
  • 'star': A star icon that will enable quick starring of a track. When the track is playing, the star icon will be replaced by the now playing icon. Use of this field is being phased out, and should soon be done from other places.
  • 'share': A share icon that will show a share popup when clicked.
Default: Depends on the type of list:
  • 'tracks': ['star', 'track', 'artist', 'time', 'album']
  • 'albums': ['album']
  • 'artists': ['artist']
height string How the height of the list will behave.
  • 'fixed': The list will keep a specific height (that you set yourself using CSS), and the list will scroll within itself, instead of using the app scroll.
  • 'dynamic': (Default) The list will increase in height when more items are added, and the list will scroll using the app scroll.
scrollElement HTMLElement If provided, and height option is set to 'dynamic', this will be the element used for detecting scrolling instead of the app scroll.
context Playlist | Album | Disc | Collection An optional context to use for playing tracks, instead of generating one.
unplayable string How to show unplayable tracks.
  • 'disabled': Show the row, but dim it to make it look disabled.
  • 'hidden': Hide the whole row.
style string The style of the list.
  • 'plain': (Default) No borders or other style around the list.
  • 'rounded': Border and rounded corners.
imageOptions Object An object that will be passed as options to the image view for images inside the list. Default: undefined.
visualOffset number An optional offset for the ordinal field. If for example 5 is provided, the first item in the list will have the number 6. Default: 0
throbber string What type of throbber to use.
  • 'none': (Default) No throbber is visible.
  • 'hide-content': Throbber is visible and content is hidden.
  • 'show-content': Throbber is visible on top of the content.
